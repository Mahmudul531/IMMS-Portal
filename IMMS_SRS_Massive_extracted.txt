1. Executive Summary & Purpose
The Infrastructure Maintenance & Management System (IMMS) is a robust enterprise platform designed to manage organizational properties, assets, infrastructure projects, maintenance work orders, and financial tracking. It provides a unified portal for administrators, engineers, technicians, contractors, and financial officers to collaborate efficiently. This expansive document details the functional, architectural, and data storage requirements of the system to ensure successful delivery by Prime Tech Solutions LTD & Infrastructure Design & Development LTD-JV.
2. Technology Stack & Architecture
2.1 Backend Architecture
• Language: Java 21• Framework: Spring Boot 4.0.5 (Spring WebMVC, Spring Security, Spring Data JPA, Spring Websocket)• Database: PostgreSQL• Spatial Data: PostGIS integrated via Hibernate Spatial & JTS Topology Suite• API Documentation: Springdoc OpenAPI (Swagger UI)• Storage/Media: Cloudinary (for scalable image hosting)• Reporting & Documents: JasperReports, Apache POI (Excel), OpenPDF, and Apache PDFBox (for PDF text extraction)
2.2 Frontend Architecture
• Core Library: React• Language: TypeScript• Build Tool: Vite• Styling: Custom CSS• Key Components: Interactive Maps (for spatial registries) and Gantt Charts (for project tracking).

3. User Roles & Dynamic Access Control
The system utilizes a hybrid authorization approach with hardcoded Roles and dynamic Permission Groups.
3.1 Primary User Roles
• ADMIN: Full system access, user management, system configuration, and overarching oversight.
• ENGINEER: Technical oversight. Responsible for creating tasks, approving technical milestones, and reviewing work orders and project progress.
• TECHNICIAN: On-ground staff. They execute assigned tasks, update work order statuses, and report on asset conditions.
• ACCOUNTS (Finance): Financial oversight. Responsible for tracking project finances, approving payment requests, and generating financial reports.
• CONTRACTOR: External vendors or agencies. They can apply for tenders, manage awarded contracts, and submit payment requests.
3.2 Dynamic Role Management (PermissionGroup)
Users are linked to a PermissionGroup entity (e.g., "Site Engineers") containing a comma-separated text string of exact privileges (e.g., CREATE_ASSET, VIEW_REPORTS, MANAGE_PROPERTIES, APPROVE_FINANCE) via the permissionGroupId. This allows granular enterprise-level access customization.
