1. Introduction
This document is the Software Design Documentation (SDD) for the Infrastructure Maintenance & Management System (IMMS). It describes the architectural choices, physical database schemas, RESTful API contracts, structural component design, and UI/UX workflows required to fulfill the SRS constraints.
2. Purpose
To translate the business requirements of HED into actionable technical blueprints for the development team. Unlike the SRS, this document focuses strictly on HOW the system is built, specifically outlining Spring Boot Beans, React Contexts, Database DDL details, and visual representations of the application flow.
3. Technical Specification
Component
Technology Choice
Backend Framework
Java 21, Spring Boot 4.0.5
Database
PostgreSQL 15+ (locLat/locLon standard columns, no PostGIS needed)
Frontend
React 19, TypeScript, Vite
Reporting Engine
JasperReports, Apache POI (Excel), OpenPDF, Apache PDFBox
Cloud Storage
Cloudinary
API Specification
Springdoc OpenAPI (Swagger UI)

6. System Architecture Design
The IMMS platform uses a standard layered MVC architectural pattern applied over a RESTful micro-service style monolith. Controller layers map DTOs via MapStruct before persisting Entities via Hibernate.
